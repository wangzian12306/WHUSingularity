# 前端页面设计

> 版本：v1.1
> 日期：2026-05-07

## 1. 概述

基于后端业务能力，定义前端需要覆盖的页面和用户交互行为。API 契约详见 [03-frontend-api-contracts.md](03-frontend-api-contracts.md)。

## 2. 页面列表

### 2.1 登录/注册页

**路由**：`/login`、`/register`
**权限**：公开

**功能**：
- 登录表单：用户名 + 密码
- 注册表单：用户名 + 密码 + 昵称（可选）
- 登录成功后跳转秒杀主页
- 注册成功后跳转登录页
- 错误提示：用户名已存在、密码错误、参数不合法

### 2.2 秒杀主页

**路由**：`/`
**权限**：需登录

**功能**：
- 商品列表：展示参与秒杀的商品（商品名、价格、库存、状态）
- 库存展示：每个商品的剩余可用库存，定时轮询刷新
- 倒计时：秒杀开始/结束倒计时
- 抢单按钮：点击后发送抢单请求，按钮禁用防重复提交
- 抢单结果：成功/失败提示（库存不足、重复抢等）
- WebMCP：页面挂载后注册 `listStock` / `snagOrder` tools，供浏览器 AI Agent 调用

### 2.3 用户中心

**路由**：`/user`
**权限**：需登录

**功能**：
- 个人信息：显示用户名、昵称、角色
- 余额展示与充值：显示当前余额，充值入口
- 我的订单：当前用户的订单列表及状态
- 退出登录：清除 token，跳转登录页

## 3. 管理页面

> 以下页面依赖管理员角色（`role=admin`），已全部实现。

### 3.1 用户管理

**路由**：`/admin/users`
**权限**：Admin

**功能**：用户列表、编辑用户信息、删除用户

### 3.2 库存管理

**路由**：`/admin/stock`
**权限**：Admin

**功能**：查看所有商品库存、初始化新商品库存、查看库存变更日志

### 3.3 订单管理

**路由**：`/admin/orders`
**权限**：Admin

**功能**：查看全部订单列表，按用户/状态筛选
