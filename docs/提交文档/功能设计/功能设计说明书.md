## 1. 文档介绍

### 1.1 编写目的

本文档对 WHUSingularity 高并发抢单系统的功能进行详细设计说明，明确系统各模块的功能需求、业务规则和界面设计。本文档将作为系统开发、测试和验收的依据，确保各参与方对系统功能有一致的理解。

### 1.2 文档范围

本文档覆盖 WHUSingularity 系统前端全部功能模块，包括：用户注册/登录、商户注册/登录、商品浏览与搜索、商品详情、高并发抢单、订单支付与查询、商户商品管理、库存管理、管理后台、WebMCP 在线调试等功能的设计说明。本文档不涉及具体的代码实现细节和部署运维方案。

### 1.3 读者对象

- 项目开发人员：了解系统功能边界和业务规则
- 测试人员：根据功能需求编写测试用例
- 项目指导教师：评审系统功能设计的完整性
- 团队成员：对系统功能达成一致理解

### 1.4 术语与缩写解释

| 术语/缩写 | 说明 |
|-----------|------|
| Actor-Slot | 自研高并发分槽路由模型，将请求按用户哈希分散到不同库存槽位，减少锁竞争 |
| 抢单/秒杀 | 高并发场景下用户抢购限量商品的业务行为 |
| Slot | 库存槽位，将商品总库存分割为多个独立库存桶，每个槽位独立锁 |
| Redis Bucket | Redis 中存储的库存桶，Key 格式为 `stock:slot-{productId}` |
| 事务消息 | RocketMQ 提供的半消息机制，保证本地事务与消息发送的原子性 |
| WebMCP | Web Model Context Protocol，浏览器端 AI Agent 工具调用协议 |
| JWT | JSON Web Token，无状态身份认证令牌 |
| Nacos | 阿里巴巴开源的服务注册与配置中心 |
| Caffeine | 高性能 Java 本地缓存库 |
| Cache-Aside | 缓存旁路模式，先查缓存，未命中查数据库并回写缓存 |

## 2. 系统概述

### 2.1 系统说明

WHUSingularity 是一个基于 Spring Cloud 微服务架构的高并发抢单（秒杀）系统。系统采用前后端分离架构，前端使用 React + TypeScript + Ant Design 构建，后端由 6 个核心微服务组成（用户服务、订单服务、库存服务、商品服务、商户服务、网关服务），通过 Nacos 实现服务注册与发现，使用 Redis 进行高并发库存扣减，RocketMQ 事务消息保证分布式一致性。

系统核心业务场景为电商秒杀：商户上架限量商品并设置库存，大量用户在同一时间抢购，系统需要在保证数据一致性的前提下处理高并发请求。

### 2.2 项目背景

在电商场景中，秒杀活动面临极高的并发压力。传统单体架构在库存扣减环节存在严重的锁竞争问题，导致系统吞吐量无法线性扩展。本项目基于自研 Actor-Slot 分槽路由模型，将库存分散到多个独立槽位，有效减少锁竞争，同时利用 Redis 原子操作和 RocketMQ 事务消息实现高性能、高可靠的抢单流程。

### 2.3 系统任务

1. 提供完整的电商秒杀业务闭环：商户管理商品 → 用户浏览抢购 → 订单支付
2. 实现高并发抢单核心链路：Actor-Slot 分槽路由 + Redis 原子减库存 + RocketMQ 事务消息
3. 提供管理后台：用户管理、库存管理、订单管理
4. 提供 WebMCP 在线调试面板：支持 AI Agent 工具调用
5. 支持容器化部署和自动扩缩容

## 3. 需求说明

### 3.1 整体需求

系统需满足以下整体需求：

1. **用户侧**：用户可以注册/登录、浏览商品、搜索商品、查看商品详情、参与抢单、支付订单、查看订单列表、充值余额
2. **商户侧**：商户可以注册/登录、管理商品（创建/编辑/删除/上下架）、管理库存、查看商户信息
3. **管理侧**：管理员可以管理用户（查看/编辑/删除）、管理库存（初始化/查看变更日志）、管理订单（查看/筛选）
4. **高并发**：抢单接口需支持高并发请求，通过分槽路由和 Redis 原子操作保证性能和数据一致性
5. **AI 集成**：通过 WebMCP 协议支持浏览器端 AI Agent 调用系统工具

### 3.2 功能需求

#### 3.2.1 需求编号规则

需求编号格式为 `MXX-FYY`，其中：
- `MXX`：模块编号，M01~M07
- `FYY`：功能编号，F01~F99

#### 3.2.2 总体模块划分

| 模块编号 | 模块名称 | 说明 |
|---------|---------|------|
| M01 | 用户模块 | 用户注册、登录、信息管理、余额充值 |
| M02 | 商户模块 | 商户注册、登录、信息管理、商品管理、库存管理 |
| M03 | 商品模块 | 商品列表展示、商品详情展示、商品搜索 |
| M04 | 订单模块 | 高并发抢单、订单支付、订单查询 |
| M05 | 库存模块 | 库存初始化、库存查询、库存变更日志 |
| M06 | 管理后台模块 | 用户管理、库存管理、订单管理 |
| M07 | WebMCP 调试模块 | 工具注册与发现、工具在线执行 |

## 4. 功能性需求

### 4.1 用户模块 (M01)

#### 4.1.1 用户注册 (M01-F01)

**功能描述**：新用户通过填写注册表单创建账号。

**业务规则**：
- 用户名为必填项，长度 4~32 个字符，仅支持字母、数字、下划线
- 密码为必填项，长度 8~64 个字符
- 昵称为可选项
- 用户名不可重复，重复时提示错误
- 注册成功后自动跳转至登录页

**输入/输出**：

| 输入项 | 类型 | 必填 | 校验规则 |
|-------|------|------|---------|
| 用户名 | string | 是 | 4~32字符，仅字母数字下划线 |
| 密码 | string | 是 | 8~64字符 |
| 昵称 | string | 否 | — |

**输出**：注册成功提示，跳转登录页；失败时显示错误信息。

#### 4.1.2 用户登录 (M01-F02)

**功能描述**：已注册用户通过用户名和密码登录系统，获取 JWT 令牌。

**业务规则**：
- 登录页提供"普通用户"和"商户"两个标签页切换
- 用户名为必填项
- 密码为必填项
- 登录成功后签发 JWT 令牌，存储于 localStorage
- 登录成功后跳转至首页
- 登录失败时显示错误提示（如密码错误、用户不存在）

**输入/输出**：

| 输入项 | 类型 | 必填 | 校验规则 |
|-------|------|------|---------|
| 用户名 | string | 是 | 非空 |
| 密码 | string | 是 | 非空 |

**输出**：JWT 令牌 + 用户基本信息；失败时显示错误信息。

#### 4.1.3 用户信息管理 (M01-F03)

**功能描述**：用户在用户中心查看个人信息。

**业务规则**：
- 展示用户名、昵称、角色（normal/admin）
- 信息从后端实时获取
- 用户角色以标签形式展示（admin 红色，normal 蓝色）

**展示字段**：

| 字段 | 说明 |
|------|------|
| 用户名 | 用户注册时的用户名 |
| 昵称 | 用户设置的昵称，未设置时显示"-" |
| 角色 | normal 或 admin，以彩色标签展示 |

#### 4.1.4 余额充值 (M01-F04)

**功能描述**：用户在用户中心充值账户余额。

**业务规则**：
- 充值金额为必填项，最小 0.01 元，精确到小数点后两位
- 充值成功后实时刷新余额显示
- 充值失败时提示错误信息
- 通过弹窗表单进行充值操作

**输入/输出**：

| 输入项 | 类型 | 必填 | 校验规则 |
|-------|------|------|---------|
| 充值金额 | number | 是 | ≥0.01，最多两位小数 |

**输出**：充值成功提示 + 余额更新；失败时显示错误信息。

### 4.2 商户模块 (M02)

#### 4.2.1 商户注册 (M02-F01)

**功能描述**：新商户通过填写注册表单创建商户账号。

**业务规则**：
- 用户名为必填项，长度 4~32 个字符
- 密码为必填项，长度 8~64 个字符
- 店铺名称为必填项
- 联系人、联系电话为可选项
- 商户注册使用独立的 JWT 认证体系
- 注册成功后跳转至登录页

**输入/输出**：

| 输入项 | 类型 | 必填 | 校验规则 |
|-------|------|------|---------|
| 用户名 | string | 是 | 4~32字符 |
| 密码 | string | 是 | 8~64字符 |
| 店铺名称 | string | 是 | 非空 |
| 联系人 | string | 否 | — |
| 联系电话 | string | 否 | — |

#### 4.2.2 商户登录 (M02-F02)

**功能描述**：已注册商户通过用户名和密码登录系统。

**业务规则**：
- 在登录页"商户"标签页进行登录
- 商户登录使用独立的 JWT 认证体系（与用户 JWT 独立）
- 登录成功后跳转至首页（商户视图）
- 商户首页显示欢迎信息和快捷操作入口

#### 4.2.3 商户信息管理 (M02-F03)

**功能描述**：商户在商户中心查看和编辑商户信息。

**业务规则**：
- 展示商户用户名、店铺名称、联系人、联系电话、地址、描述、状态
- 可编辑字段：店铺名称（必填）、联系人、联系电话、地址、描述
- 展示统计信息：商品总数、上架商品数、下架商品数
- 商户状态以标签展示（正常/禁用）

**展示字段**：

| 字段 | 说明 | 可编辑 |
|------|------|--------|
| 用户名 | 商户注册时的用户名 | 否 |
| 店铺名称 | 商户店铺名称 | 是（必填） |
| 联系人 | 商户联系人 | 是 |
| 联系电话 | 商户联系电话 | 是 |
| 地址 | 商户地址 | 是 |
| 描述 | 商户描述 | 是 |
| 状态 | 正常/禁用 | 否 |

#### 4.2.4 商品管理 (M02-F04)

**功能描述**：商户对自有商品进行创建、编辑、删除、上下架操作。

**业务规则**：
- **创建商品**：
  - 商品名称（必填）、价格（必填，≥0）、分类（必填：电子产品/服装/食品/其他）、库存数量（创建时必填，≥0）
  - 副标题、图片URL、标签（逗号分隔）为可选项
  - 创建商品时自动初始化 Redis 库存桶和 Order 服务 Slot
- **编辑商品**：可修改商品名称、副标题、价格、分类、图片URL、标签、库存数量
- **删除商品**：需二次确认，删除后不可恢复
- **上架/下架**：切换商品状态，上架后用户可见，下架后用户不可见
- 商品列表展示：商品名称、价格、分类、总库存、可用库存、状态

**输入/输出（创建商品）**：

| 输入项 | 类型 | 必填 | 校验规则 |
|-------|------|------|---------|
| 商品名称 | string | 是 | 非空 |
| 价格 | number | 是 | ≥0 |
| 分类 | string | 是 | electronics/clothing/food/other |
| 库存数量 | number | 是（创建时） | ≥0 |
| 副标题 | string | 否 | — |
| 图片URL | string | 否 | — |
| 标签 | string | 否 | 逗号分隔 |

#### 4.2.5 库存管理 (M02-F05)

**功能描述**：商户在创建和编辑商品时管理商品库存。

**业务规则**：
- 创建商品时必须设置库存数量，系统自动初始化 Redis 库存桶
- 编辑商品时修改库存数量将覆盖原有库存
- 库存数量变更时自动同步 Redis 库存桶和 Order 服务 Slot
- 可用库存为 0 时，用户端商品显示"已售罄"，不可购买

### 4.3 商品模块 (M03)

#### 4.3.1 商品列表展示 (M03-F01)

**功能描述**：用户在首页浏览所有上架商品的列表。

**业务规则**：
- 以卡片网格形式展示商品（每行 4 列，响应式适配）
- 每张商品卡片展示：商品图片、分类标签、上架状态、商品名称、副标题、价格
- 无图片时显示默认占位图
- 无商品时显示"暂无商品"空状态
- 商品列表来自 singularity-product 服务，通过 Caffeine + Redis 两级缓存加速读取
- 点击商品卡片跳转至商品详情页

**展示字段**：

| 字段 | 说明 |
|------|------|
| 商品图片 | 主图，无图时显示占位 SVG |
| 分类 | 以蓝色标签展示，未分类时显示"未分类" |
| 上架状态 | 绿色圆点+上架 / 灰色圆点+下架 |
| 商品名称 | 标题级别展示 |
| 副标题 | 最多两行，超出省略 |
| 价格 | 红色加粗展示，格式 ¥XX.XX |

#### 4.3.2 商品详情展示 (M03-F02)

**功能描述**：用户查看单个商品的详细信息，包括库存状态和抢单操作。

**业务规则**：
- 页面顶部显示商品大图（支持预览）和商品信息
- 展示商品状态标签（上架中/已下架）、分类标签、自定义标签
- 展示商品名称、副标题、价格
- 展示库存统计：可用库存、总库存、已预留数量
- 库存服务不可用时显示警告提示
- 提供"立即抢单"按钮：商品已上架且库存 > 0 时可点击，否则禁用（显示"库存不足"）
- 提供"刷新"按钮：手动刷新商品详情和库存
- 提供"返回商品列表"按钮
- 底部展示商品元信息：商品 ID、版本号、创建时间、更新时间

**展示字段**：

| 字段 | 说明 |
|------|------|

### 4.4 订单模块 (M04)

#### 4.4.1 高并发抢单 (M04-F01)

**功能描述**：用户在商品详情页或首页点击"立即抢单"按钮，发起高并发抢单请求。

**业务规则**：
- 用户必须已登录，未登录时提示"请先登录"并跳转登录页
- 商品必须处于上架状态且可用库存 > 0
- 抢单按钮点击后进入 loading 状态，防止重复提交
- 抢单请求携带 userId 和 productId
- 后端处理流程：
  1. 根据 productId 查找对应的库存 Slot
  2. 通过 RocketMQ 事务消息发送半消息
  3. 本地事务中执行 Redis Lua 脚本原子减库存
  4. 根据本地事务结果提交或回滚消息
- 抢单成功：显示订单号，跳转用户中心
- 抢单失败：显示错误信息（库存不足、系统繁忙等）
- 库存为 0 时按钮显示"已售罄"或"库存不足"并禁用

**输入/输出**：

| 输入项 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| userId | string | 是 | 当前登录用户 ID |
| productId | string | 是 | 目标商品 ID |

**输出**：成功时返回 orderId；失败时返回错误信息。

#### 4.4.2 订单支付 (M04-F02)

**功能描述**：用户在用户中心对已创建的订单进行支付。

**业务规则**：
- 订单状态为"待支付"(CREATED) 时显示"立即支付"按钮
- 支付期间按钮进入 loading 状态，其他支付按钮禁用
- 支付成功后订单状态变为"已支付"(PAID)
- 支付失败时显示错误提示
- 支付成功后自动刷新用户余额和订单列表

**输入/输出**：

| 输入项 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| orderId | string | 是 | 订单 ID |
| userId | string | 是 | 当前用户 ID |

#### 4.4.3 订单查询 (M04-F03)

**功能描述**：用户在用户中心查看自己的订单列表。

**业务规则**：
- 展示当前用户的所有订单
- 订单列表字段：订单号（截断显示）、商品信息、状态、创建时间
- 订单状态以标签展示：已支付(绿色)、已取消(红色)、待支付(蓝色)
- 待支付订单显示"立即支付"按钮
- 支持分页浏览

**展示字段**：

| 字段 | 说明 |
|------|------|
| 订单号 | 截取前 12 位 + "..." |
| 商品 | Slot ID |
| 状态 | 已支付/已取消/待支付 + 支付按钮 |
| 创建时间 | 完整时间戳 |

| 页面 | 路由 | 权限 | 功能概述 |

| 用户中心 | `/user` | 用户 | 个人信息 + 余额充值 + 订单列表 + 支付 |
| 商户中心 | `/merchant/center` | 商户 | 商户信息编辑 + 商品统计 |

| 页面 | 路由 | 权限 | 功能概述 |

| 用户管理 | `/admin/users` | 管理员 | 用户列表 + 编辑 + 删除 |
| 库存管理 | `/admin/stock` | 管理员 | 库存列表 + 初始化 + 变更日志 |
| 订单管理 | `/admin/orders` | 管理员 | 订单列表 + 用户/状态筛选 |

### 5.2 界面关系

#### 5.2.1 前台页面流转

```
登录页 (/login)
  ├── 用户登录成功 → 首页 (/)
  │     ├── 点击商品卡片 → 商品详情 (/products/:id)
  │     │     └── 抢单成功 → 用户中心 (/user)
  │     ├── 导航栏"用户中心" → 用户中心 (/user)
  │     │     ├── 充值余额（弹窗）
  │     │     └── 支付订单（行内按钮）
  │     └── 导航栏"WebMCP" → WebMCP 调试 (/playground)
  └── 商户登录成功 → 首页（商户视图）(/)
        ├── "管理商品" → 商品管理 (/merchant/products)
        │     ├── 创建商品（弹窗）
        │     ├── 编辑商品（弹窗）
        │     ├── 上架/下架（行内按钮）
        │     └── 删除商品（二次确认）
        └── "商户设置" → 商户中心 (/merchant/center)
              └── 编辑商户信息（弹窗）

注册页 (/register)
  ├── 用户注册成功 → 登录页 (/login)
  └── 商户注册成功 → 登录页 (/login)
```

#### 5.2.2 后台页面流转

```
导航栏"管理页"下拉菜单（仅管理员可见）
  ├── 用户管理 (/admin/users)
  │     ├── 编辑用户（弹窗：昵称/角色/余额）
  │     └── 删除用户（二次确认）
  ├── 库存管理 (/admin/stock)
  │     ├── 初始化库存（弹窗：商品ID/数量）
  │     └── 变更日志（弹窗：变更记录表格）
  └── 订单管理 (/admin/orders)
        └── 筛选查询（用户ID/状态 + 查询/重置）
```
